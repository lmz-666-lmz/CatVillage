智宠通译（猫村APP）- 系统架构与基础设施设计文档
文档版本：V1.0
架构负责人：明哲

1. 系统概述
本项目是一个高度模块化的猫咪社交与健康管理综合性平台，采用标准的前后端分离架构设计。系统旨在通过数字化手段统一管理宠物健康、分析宠物情绪，并为用户提供围绕宠物生活的社交互动与智能化（AI）辅助服务。

2. 技术栈全景
2.1 前端应用层
前端基于极速构建和强类型校验的现代 Web 工程化标准搭建：

核心框架：Vue 3 组合式 API (Composition API) 搭配 TypeScript 提供严格的类型安全

构建与脚手架：Vite 提供毫秒级的冷启动和高效的热更新体验

状态管理：Pinia 用于处理全局应用状态，包括用户身份令牌、猫咪档案 CRUD 操作以及全局猫咪上下文的切换同步

UI 与交互：引入 Vant 作为核心移动端组件库，保障界面的响应式与用户体验

质量保障体系：

代码规范：集成了 ESLint (含 oxlint)、Prettier 确保团队代码风格一致性

自动化测试：配置了 Vitest 用于单元测试，Playwright 负责端到端 (E2E) 测试的闭环覆盖

2.2 后端服务层
后端采用高性能的异步 Python Web 技术栈构建，侧重于高并发处理与 AI 模型集成：

核心框架：FastAPI 框架驱动，提供自动化的 OpenAPI 文档生成与原生的异步路由支持

持久化与数据访问：基于 SQLAlchemy 构建对象关系映射 (ORM) 层，底层通过 PyMSSQL 驱动连接 SQL Server 数据库引擎

安全与认证：利用 passlib[bcrypt] 进行强密码哈希加密，配合 python-jose[cryptography] 实现无状态的 JWT 身份验证与授权拦截

2.3 AI 算法与多媒体处理层
作为平台的核心壁垒，系统在后端专门集成了算法处理模块：

大模型集成：通过 openai SDK 接入大语言模型，驱动平台的“AI 助理”提供智能问答与紧急帮助

音频与情绪识别：依托 librosa 和 soundfile 进行猫咪声音等音频特征的提取，结合 scikit-learn 和 numpy 进行机器学习分析，实现猫咪情绪的智能识别与统计

3. 核心业务模块划分
为保证系统的高内聚低耦合，后端的路由结构被划分为以下独立微服务模块：

Auth (/api/v1/auth)：提供用户注册、登录、Token 签发与鉴权服务

Pet Profiles (/api/v1/pet_profiles)：维护宠物主数据，包含档案的增删改查

Emotions (/api/v1/emotions)：负责处理音频上传、调用识别算法并持久化情绪记录

AI Assistant (/api/v1/ai_assistant)：承接用户与智能助手的对话会话及历史记录管理

Health (/api/v1/health)：处理宠物的健康指标跟踪与数据记录

Social (/api/v1/social)：支撑社区生态，包括图文动态发布、评论与点赞互动

Message (/api/v1/message)：处理好友关系网络以及用户间的即时消息通讯

4. 基础设施与工程化特性
4.1 全局上下文切换机制
系统在前端架构中设计了独特的“当前猫咪状态管理”机制。当用户在多只宠物档案间进行切换时，应用能通过 Pinia 状态流转确保其他模块（如健康记录、情绪数据）自动触发更新操作，无需重载页面。

4.2 环境解耦与 Mock 数据服务
为打破前后端开发依赖，前端集成了完整的数据 Mock 拦截器：

构建了包含所有业务功能（猫咪列表、消息、社交动态等）的模拟数据集

通过环境变量 VITE_ENABLE_MOCK 实现真实 API 调用与 Mock 数据的无缝动态切换

Mock 数据结构严格对齐后端 OpenAPI 接口规范，保障了联调阶段的数据一致性

4.3 跨域与网络通信
针对前后端分离带来的同源策略限制，后端配置了灵活的 CORS 中间件，开发环境下默认放行前端开发服务器（如 http://localhost:5173），并支持通过读取 CORS_ORIGINS 环境变量进行生产环境白名单的动态下发，确保安全性与灵活性。
